Cooperation for the progress of Moodle from universities

Within the framework of the NextGeneration European funds with the UNIDIGITAL program, the universities of Valladolid, Complutense de Madrid, País Vasco/EHU, León, Salamanca, Illes Balears, Valencia, Rey Juan Carlos, La Laguna, Zaragoza, Málaga, Córdoba, Extremadura, Vigo, Las Palmas, and Burgos created in October 2022 an inter-university group to develop new tools to improve Moodle-based virtual campuses.

Objectives

To maximize future sustainability, the projects have been designed with the fundamental goal of maximizing usefulness for the Moodle user community. The components developed in this project must have the following characteristics:
Open Source

Published as Open Source and available in the official Moodle directory: they must have sufficient quality guarantees for publication and support. The goal is to create a user community and all external collaborators are welcome.

General Utility

They must be reusable by other institutions without code modifications. Specific functionality should be controlled through configuration. Dependencies between components should be optional and implemented through sub-plugins (for Moodle developments) or REST APIs (for external systems). External dependencies should adhere to technological standards whenever they exist.

Simple Architecture

They must follow Moodle development specifications and avoid mixing programming languages or unnecessarily including other external platforms that may hinder free deployment. External platforms, if needed, will also be Open Source. The main languages will be PHP and JavaScript.

Flexible

They must include the most extensive and configurable use cases possible.

User-Friendly

Designed to handle complexity progressively, so that the most frequent functionality is offered in a simple interface and specific options are expanded or revealed when necessary. Fully adheres to Moodle's API and interface design style.

Integrable

The data model and reusable operations will be exposed through Moodle's Webservices architecture to allow integration with other systems.
